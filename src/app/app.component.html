Introducción

¿Qué son las GANs?

Las redes adversas generativas (GAN) son una arquitectura de aprendizaje profunda innovadora, que ha ganado mucha atención en los últimos años debido a su capacidad para generar datos sintéticos altamente realistas, que son difíciles de distinguir de los datos reales. A diferencia de otros enfoques de aprendizaje automático que se centran en el reconocimiento de patrones o clasificación, las redes generativas se enfocan en la creación de nuevos datos. Las GAN fueron presentadas en 2014 por Ian Goodfellow y sus colegas, y constan de dos modelos de redes neuronales: el generador y el discriminador.

Arquitectura de una GAN

Vector Ruido

Vector ruido o comúmente llamado vector aleatorio en un espacio latente. Se genera a través de una función de distribución (comúnmente una distribución N∼(0,1)) y sirve como una entrada para la red generadora. Dicho vector pasará por capas, pesos y activaciones para convertirse en una imagen de (“X” x “Y”) píxeles.

Dataset Real

El dataset real se utiliza durante el entrenamiento en mini-batches, no todo de una vez. En cada iteración, se toma un lote de imágenes reales del dataset y se compara con otro lote de imágenes falsas generadas por el generador a partir de vectores de ruido. Este enfoque por lotes es eficiente, permite usar descenso de gradiente mini-batch y mejora la estabilidad del entrenamiento.

Red Generadora

Red neuronal que genera nuevos datos a partir del vector ruido. Su objetivo es crear ejemplos tan realistas que engañen al discriminador.

Capa Densa Capa De Activación No Lineal Capa De Normalización Capa de Activación de Salida Capa Reshape

Red Discriminadora

Red neuronal que se encarga de clasificar si un dato es real o generado. Aprende a detectar qué ejemplos provienen del dataset real y cuáles son falsos.

Capa Flatter Capa Densa Capa De Activación No Lineal Capa de Salida

Proceso de Entrenamiento

Inicialización

Se inicia con la construcción de dos redes neuronales profundas: el Generador (G) y el Discriminador (D). Ambas redes parten con pesos aleatorios y sin conocimiento previo del dominio, estableciendo así la base para su entrenamiento conjunto.

El generador crea datos sintético

El Generador toma como entrada un vector de ruido aleatorio, como 100 valores con distribución normal, y lo transforma, a través de su red, en una muestra sintética como una imagen, audio o texto. Al inicio, estas salidas carecen de coherencia, pero con el entrenamiento, el Generador aprende a producir datos cada vez más similares a los reales observados por el Discriminador pese a estar generadas desde ruido.

El discriminador analiza los datos

El Discriminador funciona como un evaluador binario que recibe tanto datos reales del conjunto de entrenamiento como datos sintéticos generados por el Generador. Su objetivo es clasificarlos correctamente, asignando una probabilidad alta (cercana a 1) si considera que el dato es real, y baja (cercana a 0) si lo percibe como falso

Obtención de funciones de pérdida

Las funciones de pérdida se obtienen evaluando las salidas del Discriminador (clasificación binaria). A partir de estos valores se calcula la pérdida del Discriminador (LD), que mide su capacidad para distinguir correctamente entre ambas clases. Por su parte, la pérdida del Generador (LG) se calcula considerando qué tan cerca estuvo de lograr que el Discriminador clasificara sus salidas falsas como reales, es decir, que asignara valores cercanos a 1 a datos generados. Estas funciones permiten cuantificar el desempeño de cada red en cada iteración del entrenamiento.

Optimización mediante retropropagación

Una vez obtenidas las funciones de pérdida, se utiliza retropropagación del error junto con algoritmos de optimización como el descenso del gradiente estocástico (SGD) para ajustar los pesos internos de ambas redes. El Generador modifica sus parámetros para generar datos más convincentes, mientras que el Discriminador refuerza su capacidad para identificar falsificaciones. Este proceso se repite a lo largo de múltiples épocas, permitiendo que ambas redes mejoren progresivamente y de forma conjunta.

Una GAN se considera que "ya sirve" o ha alcanzado un buen punto de convergencia cuando el discriminador tiene una precisión cercana al 50%, lo que indica que no puede diferenciar con certeza entre datos reales y generados.

Ventajas

Generación realista de datos

Las GANs pueden generar imágenes, textos o sonidos sintéticos que se asemejan mucho a datos reales

No requieren etiquetado

Solo necesitan datos reales sin necesidad de etiquetas (aprendizaje no supervisado)

Aplicables a múltiples dominios

Se usan en imágenes, audio, texto, videojuegos, arte y más. Las personas pueden experimentar con ideas innovadoras y plasmarlas antes de invertir tiempo y recursos en la producción física

Capacidad de aprender distribuciones complejas

Las GANs pueden modelar distribuciones de datos muy complejas y de alta dimensión

Desafíos y Limitaciones

Dificultad de entrenamiento

El entrenamiento de las GAN puede ser inestable y requiere una cuidadosa selección de hiperparámetros

Modo colapso

En algunos casos, el generador puede converger a un modo de colapso, donde genera datos que son muy similares entre sí

Evaluación de la calidad de las muestras generadas

No hay una métrica única y objetiva que mida cuán “bueno” es un generador

Interpretabilidad

Puede ser difícil entender por qué las GAN están generando ciertos tipos de datos

No garantizan diversidad

Aunque producen ejemplos realistas, pueden no representar toda la variedad de los datos reales

Costo computacional elevado

Entrenar GANs profundas requiere GPUs potentes y muchos recursos

Complejidad en la implementación

Diseñar y ajustar una GAN adecuada puede requerir mucha experimentación y experiencia previa.
